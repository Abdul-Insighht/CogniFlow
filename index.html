Project Overview

Neutral

Complete local JavaScript system

CogniFlow helps students decide what to study next when work feels scattered or stressful.

Students add tasks, notes, study logs, and a short check-in. CogniFlow reads the workload, detects stress/focus signals, finds relevant saved notes, and builds a practical plan that the student can accept, edit, or reject.

Overwhelmed sample output
  1. Work on "Revise DBMS normalization" for 25 minutes only.
  2. Open memory note "DBMS revision checklist" before starting.
  3. Hide non-urgent tasks until the first block is complete.

Problem

Students often have quizzes, assignments, notes, and personal pressure at the same time. The hard part is deciding the next action.

Input

The system accepts a check-in sentence, academic tasks, course notes, and study history logs.

AI Logic

It uses local keyword/NLP signals, task urgency scoring, note retrieval, and weak-topic detection. No paid API is required.

Output

It returns a cognitive state, confidence, detected signals, relevant notes, and an adaptive plan.

Why Helpful

A stressed student gets a smaller first step instead of a long generic to-do list.

Privacy

All demo data is stored locally in the browser using localStorage. There is no server upload.

Non-Technical Demo

Use this flow to explain CogniFlow to any non-technical person in 2-3 minutes.

1. Student writes a check-in

Example: I feel stressed because my DBMS quiz is tomorrow and normalization is weak.

2. Student adds tasks

Example tasks include DBMS revision, AI assignment, and OS reading with priorities and due dates.

3. Student saves notes

The memory stores useful notes like DBMS weak topics and revision checklists.

4. CogniFlow analyzes the situation

It detects stress words, urgent deadlines, active task count, and weak topics from study history.

5. CogniFlow gives a plan

The output is a small action plan, relevant note suggestions, confidence score, and reasons.

6. Student stays in control

The student can accept, edit, or reject the recommendation.

    Accepted Plan

    Responsible AI Controls

    • Local browser storage for demo data.
    • No medical diagnosis or therapy claims.
    • Transparent stress signals and confidence score.
    • Student approval required before a plan is adopted.
    • Delete, reset, export, and import controls for user data.

    Known Limits

    • Keyword and pattern-based classifier can be wrong.
    • Recommendations are decision support, not commands.
    • Demo uses synthetic examples and user input only.
    • Users in crisis should contact trusted people or local emergency support.
